# Business Entities

Folder này mô tả các thực thể business chính mà đội vận hành làm việc cùng.

## Cách đọc

- Đọc `project.md` để hiểu đơn vị cấu hình và quản trị vận hành.
- Đọc `issue.md` để hiểu đối tượng trung tâm mà CIS đang quản lý.
- Đọc `comment.md`, `attachment.md` để hiểu các đối tượng con đi kèm issue.
- Đọc `mapping.md`, `anomaly.md` để hiểu các lớp dữ liệu quyết định issue có thể đi tiếp hay không.

## Nội dung chính

1. [project.md](project.md)
2. [issue.md](issue.md)
3. [comment.md](comment.md)
4. [attachment.md](attachment.md)
5. [mapping.md](mapping.md)

## Câu hỏi mà layer này trả lời

- Đội vận hành đang quản lý object nào.
- Mỗi object tồn tại để phục vụ quyết định business gì.
- Khi object có vấn đề thì thường block ở bước nào.
