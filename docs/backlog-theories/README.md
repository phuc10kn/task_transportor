# Backlog Theories

## Mục đích

`docs/backlog-theories/` giữ các ghi chú phân tích, review, candidate theory hoặc proposal documentation chưa nên coi là canonical.

Folder này dùng cho nội dung kiểu:

- review một tài liệu tổng hợp trước khi loại bỏ hoặc promote;
- phân loại kiến thức nào có thể promote vào docs canonical;
- giữ ý tưởng tốt nhưng chưa đủ chín để thành rule;
- ghi lại nội dung không nên áp dụng để tránh lặp lại cùng tranh luận.

## Không phải source of truth

Folder này không thay thế:

```text
docs/meta/
docs/app/
docs/theories/
docs/app_technical/
docs/app/10-decisions/
```

Khi nội dung trong backlog đủ ổn định, promote vào đúng home canonical và giữ link/provenance ở đây.

## Nội dung hiện có

| File | Nội dung |
| --- | --- |
| [more-context-review.md](more-context-review.md) | Review file `more_context.md` đã bị loại bỏ: phần đã promote, phần heuristic, phần không nên promote |
| [more-context-unpromoted.md](more-context-unpromoted.md) | Nội dung chưa promote từ `more_context.md`, giữ làm candidate backlog |
