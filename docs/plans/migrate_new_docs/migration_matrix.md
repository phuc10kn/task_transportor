# Migration matrix

File này là sổ cái bắt buộc cho migration từ `docs_legacy/` sang `docs/`.

Không được xóa `docs_legacy/` nếu file này còn dòng có cột `status` bằng `migrate`, `merge`, `defer` hoặc `keep-temporary`.

## Quy tắc cập nhật

- Mỗi file trong `docs_legacy/` phải có ít nhất một dòng matrix.
- Nếu một file chứa nhiều loại tri thức, tách thành nhiều dòng theo `knowledge_slice` trong phase owner tương ứng.
- Không dùng trạng thái cấp folder thay cho trạng thái cấp file nếu folder có nội dung không đồng nhất.
- Mỗi dòng `migrate` hoặc `merge` phải có `destination`.
- Mỗi dòng `discard` hoặc `superseded` phải có `reason`.
- Mỗi dòng `defer` phải có `review_question`.
- Mỗi dòng `keep-temporary` phải có `deadline_phase`.
- Khi hoàn tất một phase, cập nhật cột `phase_owner`, `status`, `destination`, `evidence` bằng kết quả đọc sâu thật.

## Status hợp lệ

| Status | Ý nghĩa |
| --- | --- |
| `migrate` | Nội dung còn là source of truth và chưa có trong `docs/`. |
| `merge` | Nội dung đã có một phần trong `docs/`, nhưng legacy còn chi tiết đúng cần nhập thêm. |
| `superseded` | Nội dung đã được thay thế bởi docs mới. |
| `discard` | Scratch, lỗi thời, hoặc không còn liên quan. |
| `defer` | Cần người review quyết định trước khi xử lý. |
| `keep-temporary` | Cần giữ tạm để đối chiếu trong phase sau. |
| `migrated` | Đã migrate xong và đã có evidence. |
| `merged` | Đã merge xong phần còn sống vào `docs/` và đã có evidence. |
